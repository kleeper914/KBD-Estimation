🎯 核心创新点

【1. 温度参数可控的知识蒸馏】
   问题：传统蒸馏无法精细控制发收端知识差异
   解决：引入温度参数T ∈ [1.0, 10.0]
   效果：可生成任意差异程度的模型组合
   
   T = 2.0  → 硬蒸馏 (差异大) ───────────┐
   T = 4.0  → 平衡   (差异中) ─ 推荐 ───┤ 知识差异程度
   T = 8.0  → 软蒸馏 (差异小) ───────────┘

【2. 灵活的部分蒸馏】
   ✓ full     - 完整蒸馏(发+收都蒸馏)
   ✓ encoder  - 仅发端蒸馏
   ✓ decoder  - 仅收端蒸馏
   
   应用：精确研究发端或收端的知识差异

【3. 自动化混合模型生成】
   功能：create_hybrid_model()
   作用：自动组合原始和蒸馏的模型权重
   结果：原始Encoder + 蒸馏Decoder (或反之)
   
   示意：
   ┌─────────────────────────────────┐
   │  原始教师模型                   │
   ├─────────────────────────────────┤
   │  Encoder    Channel    Decoder  │
   └─────────┬──────────────┬────────┘
             │              │
          [保留]         [蒸馏]
             │              │
             └──────┬───────┘
                    ▼
        ┌──────────────────────┐
        │   混合模型           │
        │ (发收端知识差异系统) │
        └──────────────────────┘

【4. 完整的分析工具链】
   semantic_analysis.py 提供：
   - KL散度      : 输出分布差异
   - 表示距离    : 中间层特征差异
   - 一致性      : 预测结果相似度
   - 综合分数    : 总体知识差异评分
   
   用途：科学量化和对比不同蒸馏模型

🔧 主要参数说明

【核心参数】
  --temperature              (float, 默认4.0)
    ├─ 范围: 1.0 - 10.0
    ├─ 建议: 2.0(硬)/4.0(平衡)/8.0(软)
    └─ 作用: 控制知识的软硬程度
  
  --distill-weight          (float, 默认0.7)
    ├─ 范围: 0.3 - 0.9
    ├─ 作用: 蒸馏损失的权重比例
    └─ 建议: 0.7(平衡蒸馏)
  
  --distill-part            (str, 默认'full')
    ├─ 选项: full/encoder/decoder
    └─ 作用: 选择蒸馏的模型部分
  
  --create-hybrid           (bool flag)
    ├─ 作用: 是否生成混合模型
    └─ 位置: 原始Encoder + 蒸馏Decoder

【模型参数】
  --student-layers          (int, 默认2) - 学生模型层数
  --d-model                 (int, 默认128) - 模型维度
  --num-heads              (int, 默认8) - 注意力头数
  --batch-size             (int, 默认128) - 批次大小
  --epochs                 (int, 默认80) - 训练轮数
  --lr                     (float, 默认1e-4) - 学习率
